🔤 Categoría 1 — Tipografía
🔤
font-family — Tipo de fuente
font-family: 'Nombre', genérica;
▼

¿Qué hace? Define la familia tipográfica (fuente) que se aplicará al texto del elemento.

Sintaxis: Se escriben los nombres de fuente separados por comas. Si el nombre tiene espacios, va entre comillas. Al final siempre se coloca una fuente genérica de respaldo (sans-serif, serif, monospace) por si la principal no carga.

/* Fuente del sistema con respaldo */
body {
  font-family: 'Georgia', serif;
}

/* Google Font con respaldo (buena práctica) */
h1 {
  font-family: 'Roboto', 'Helvetica Neue', sans-serif;
}

/* Importar Google Font al inicio del CSS */
@import url('https://fonts.googleapis.com/css2?family=Roboto');
💡

Las comillas en el nombre de la fuente son buena convención, especialmente si el nombre tiene espacios (ej: 'Open Sans'). La fuente genérica al final es el seguro si la principal falla.

⚠️

Propiedades inválidas que NO existen: text-font, font-type. Solo existe font-family.

⚖️
font-weight — Grosor del texto
font-weight: bold | 400 | 700;
▼

¿Qué hace? Controla el grosor o "peso" visual de los caracteres del texto.

ValorSignificadoEquivalente
100–300Texto delgado (thin, light)light
400Texto normalnormal
700Texto en negritabold
900Texto extra gruesoblack
p     { font-weight: 400; }    /* normal */
strong { font-weight: bold; }   /* = 700 */
h1    { font-weight: 700; }    /* = bold */
⚠️

No existe font-style: bold ni text-weight. El grosor siempre es font-weight. font-style es para cursiva (italic).

✏️
text-decoration & text-align
subrayado, tachado, alineación
▼

text-decoration añade o elimina adornos visuales al texto.

text-align controla la alineación horizontal del texto dentro de su bloque.

/* Eliminar subrayado de enlace */
a {
  text-decoration: none;
}

/* Subrayar texto */
.subrayado {
  text-decoration: underline;
}

/* Centrar texto en su contenedor */
.centrado {
  text-align: center;  /* left | right | center | justify */
}
💡

Para un enlace elegante que además esté centrado, necesitas dos propiedades: text-decoration: none; + text-align: center;

🎨 Categoría 2 — Color y Fondo·
🎨
color vs background-color
texto vs fondo del elemento
▼

color define el color del texto (y decoraciones de texto) del elemento.

background-color pinta el área de fondo que ocupa el elemento según su modelo de caja.

.tarjeta {
  color: #ffffff;            /* texto blanco */
  background-color: #1a1a2e; /* fondo azul oscuro */
}

/* Formatos de color aceptados */
color: red;              /* nombre */
color: #FF5733;         /* hexadecimal */
color: rgb(255,87,51);   /* RGB */
color: rgba(255,87,51,0.5); /* RGBA con opacidad */
💡

El hexadecimal siempre empieza con # seguido de 6 caracteres (o 3 en formato abreviado). Ejemplo válido: #FF5733.

🌈
linear-gradient — Fondos degradados
background-image: linear-gradient(...);
▼

Los degradados CSS son imágenes generadas dinámicamente, por eso se aplican con background-image, no con background-color.

/* De arriba a abajo: rojo → azul */
div {
  background-image: linear-gradient(to bottom, red, blue);
}

/* De izquierda a derecha */
div {
  background-image: linear-gradient(to right, #ff0000, #0000ff);
}

/* En diagonal a 45 grados */
div {
  background-image: linear-gradient(45deg, red, blue);
}

/* Forma abreviada también válida */
div {
  background: linear-gradient(to right, red, blue);
}
⚠️

background-color: linear-gradient() NO funciona. Los degradados son imágenes → siempre background-image o background.

📦 Categoría 3 — Modelo de Caja·
📐
margin & padding — El Box Model
espacio exterior vs interior
▼

El modelo de caja describe cómo cada elemento HTML ocupa espacio. Tiene 4 capas concéntricas:

margin (exterior)
border
padding (interior)
contenido
/* Valores individuales */
.caja {
  margin-top:    20px;
  margin-right:  10px;
  margin-bottom: 20px;
  margin-left:   10px;
}

/* Shorthand: top  right  bottom  left  (sentido horario) */
.caja { margin: 20px 10px 20px 10px; }

/* Shorthand con 2 valores: vertical  horizontal */
.caja { margin: 20px 10px; }

/* Centrar horizontalmente un elemento con ancho definido */
.centrado { margin: 0 auto; max-width: 800px; }
💡

Regla mnemotécnica para los 4 valores: TRouBLe → Top Right Bottom Left. Siempre en sentido horario.

⬜
border, border-radius & box-shadow
bordes, esquinas y sombras
▼ 
/* border shorthand: grosor estilo color */
.caja {
  border: 2px solid red;
}

/* border-radius: redondear esquinas */
.redondeado { border-radius: 10px; }
.circulo    { border-radius: 50%; }   /* cuadrado → círculo */

/* box-shadow: offset-x  offset-y  blur  color */
.sombra {
  box-shadow: 5px 10px 15px rgba(0,0,0,0.5);
  /*          ^^^  ^^^   ^^^
             horiz vert  BLUR ← 3er valor es desenfoque */
}
💡

En box-shadow: 1° offset-x, 2° offset-y, 3° blur (desenfoque), 4° spread (expansión opcional), 5° color. El tercer valor es el blur.

📏
max-width — Diseño responsivo
max-width: 100% en imágenes
▼

max-width establece el ancho máximo que puede alcanzar un elemento. Si el contenedor es más pequeño, el elemento se reduce. Si es más grande, el elemento no supera su max-width.

/* Imagen responsiva (buena práctica) */
img {
  max-width: 100%;   /* ✅ Se adapta, nunca se estira */
  height: auto;      /* mantiene proporciones */
}

/* width: 100% fuerza siempre el 100% del contenedor */
img {
  width: 100%;        /* ⚠️ Puede pixelar si el contenedor es grande */
}
💡

max-width: 100% es la mejor práctica para imágenes porque: si el contenedor es más pequeño, la imagen se reduce; si es más grande, la imagen mantiene su tamaño original sin pixelarse.

⚙️ Categoría 4 — Display y Flexbox·
⬛
display: flex — Contenedor flexible
organización horizontal por defecto
▼

Al aplicar display: flex a un contenedor, sus hijos directos se convierten en elementos flex organizados en fila horizontal por defecto.

.contenedor {
  display: flex;               /* activa flexbox */
  flex-direction: row;         /* por defecto: fila */
  justify-content: flex-start; /* por defecto: inicio */
  align-items: stretch;        /* por defecto: estirar */
}

justify-content — distribuye los elementos en el eje horizontal:

space-between:

Item 1
Item 2
Item 3

center:

Item 1
Item 2
Item 3

space-around:

Item 1
Item 2
Item 3
/* Los valores más usados de justify-content */
justify-content: center;        /* agrupa en el centro */
justify-content: space-between; /* extremos pegados a bordes */
justify-content: space-around;  /* espacio alrededor de cada uno */
justify-content: space-evenly;  /* espacio perfectamente igual */
🔁
flex-wrap & gap
salto de línea y espacio entre elementos
▼

flex-wrap: wrap permite que los elementos flex salten a una nueva línea cuando no hay suficiente espacio en el contenedor. Por defecto es nowrap (todo en una sola fila).

gap define el espacio entre los elementos hijos, tanto en filas como columnas. Solo actúa entre elementos, no en los bordes exteriores.

.grid-tarjetas {
  display: flex;
  flex-wrap: wrap;     /* salta a nueva fila si no hay espacio */
  gap: 20px;           /* espacio entre items (filas y columnas) */
  gap: 16px 24px;    /* gap-row gap-column (también válido) */
}

/* Ventaja de gap vs margin: */
/* margin afecta los 4 lados → bordes exteriores indeseados */
/* gap solo actúa ENTRE elementos → diseño más limpio */
✨ Categoría 5 — Interactividad ·
🖱️
Pseudoclases de estado
:hover · :active · :focus · :visited
▼

Las pseudoclases permiten aplicar estilos según el estado interactivo del elemento.

:hover
Se activa cuando el cursor del mouse está encima del elemento. La más usada para efectos interactivos.
:active
Se activa mientras el usuario mantiene el clic presionado sobre el elemento.
:focus
Se activa cuando el elemento recibe el foco, generalmente por teclado (Tab) o clic en inputs.
:visited
Se aplica a enlaces que el navegador registra que el usuario ya ha visitado previamente.
.boton {
  background-color: #5c6bc0;
}
.boton:hover   { background-color: #7986cb; }
.boton:active  { background-color: #3949ab; }
.boton:focus   { outline: 2px solid #90caf9; }

a:visited { color: purple; }  /* convención: morado */
🎬
transform & transition
transformar y animar suavemente
▼

transform aplica transformaciones visuales al elemento: rotar, escalar, mover, sesgar. Se pueden concatenar múltiples funciones separadas por espacio.

transition suaviza los cambios entre estados. Se coloca en el estado base (no en el :hover). Necesita un disparador para ejecutarse.

Pasa el cursor sobre cada caja para ver el efecto:

rotate(45deg)
scale(1.4)
translateX(12px)
border-radius:50%
/* transform: concatenar funciones con espacio */
.elemento {
  transform: rotate(45deg) scale(2);
}

/* transition: propiedad  duración  curva */
.boton {
  background-color: blue;
  transition: background-color 0.5s ease;
}
.boton:hover {
  background-color: red;  /* se anima con 0.5s de duración */
}

/* SIN transition → cambio instantáneo (brusco) */
/* CON transition → cambio fluido y suave */

/* Importante: transform sin :hover no se activa solo */
.boton       { transform: scale(1); transition: transform 0.3s; }
.boton:hover { transform: scale(1.1); } /* :hover dispara la transformación */
⚠️

Si defines transform: scale(1.1) en el estado base sin un disparador como :hover, el elemento simplemente estará escalado permanentemente pero no se verá ninguna animación al interactuar.

Los 5 conceptos fundamentales que no puedes olvidar para el cuestionario.

01

Cada propiedad tiene su función exacta

font-family = fuente, font-weight = grosor, font-size = tamaño. Confundirlas es el error más frecuente.

02

Box Model: margin exterior, padding interior

margin = espacio fuera del borde. padding = espacio dentro. Los 4 valores van en sentido horario: top → right → bottom → left.

03

Flexbox: fila horizontal por defecto

justify-content controla el eje horizontal. flex-wrap: wrap permite saltos de línea. gap es mejor que margin para separar items.

04

Degradados son imágenes, no colores

Usar background-image: linear-gradient(), nunca background-color. Los transform se concatenan con espacio: rotate(45deg) scale(2).

05

Interactividad: pseudoclases + transition

:hover detecta el cursor. La transition va en el estado base para suavizar cambios. transform sin disparador como :hover no genera animación visible.

🔤 Tipografía

  • ✦ Google Fonts: importar con @import url(), aplicar con font-family: 'Roboto', sans-serif;
  • ✦ Negrita: font-weight: bold o font-weight: 700 son equivalentes.
  • ✦ Subrayado: text-decoration: underline lo añade, none lo elimina.
  • ✦ Trampa: font-style es para cursiva (italic), NO para negrita.

🎨 Color y Fondo

  • ✦ color = texto (primer plano). background-color = fondo del elemento.
  • ✦ Los degradados son imágenes CSS → usar background-image: linear-gradient()
  • ✦ Trampa: background-color: linear-gradient() no funciona.

📦 Modelo de Caja

  • ✦ margin = exterior (entre elementos), padding = interior (entre contenido y borde).
  • ✦ Orden de 4 valores: top → right → bottom → left (TRouBLe).
  • ✦ En box-shadow, el 3er valor es el blur (desenfoque).
  • ✦ border-radius: 50% en elemento cuadrado = círculo perfecto.

⚙️ Flexbox

  • ✦ display: flex → hijos en fila horizontal por defecto (flex-direction: row).
  • ✦ justify-content: space-between → 1° pegado a la izquierda, último a la derecha.
  • ✦ flex-wrap: wrap → permite múltiples filas cuando no hay espacio.
  • ✦ gap vs margin: gap solo actúa entre elementos, sin afectar bordes exteriores.

✨ Interactividad

  • ✦ :hover = cursor encima. :active = clic sostenido. :focus = teclado. :visited = ya visitado.
  • ✦ La transition va en el estado base, no en :hover. Necesita un disparador para ejecutarse.
  • ✦ Trampa: transform sin :hover no crea animación al interactuar.

respuestas correctas en opción múltiple
Tipografía
¿Cuál de las siguientes reglas aplica correctamente una fuente de Google Fonts con fuente de respaldo?
Correcto: B. La propiedad es font-family. Las comillas son buena práctica y la fuente genérica de respaldo (sans-serif) se separa con coma.
Color y Fondo
¿Por qué los degradados CSS deben usarse con background-image en lugar de background-color?
Correcto: C. linear-gradient() genera una imagen de forma dinámica, por eso se asigna a background-image. background-color solo acepta valores de color plano.
Modelo de Caja
Dada la regla margin: 10px 20px 30px 40px, ¿cuál es el valor del margen izquierdo?
Correcto: D. El orden de 4 valores sigue el sentido horario: top=10px, right=20px, bottom=30px, left=40px. Mnemotécnica: TRouBLe.
Flexbox
¿Cuál es la ventaja principal de usar gap en un contenedor flex en lugar de margin en los elementos hijos?
Correcto: B. gap actúa como "canaleta" entre elementos. El margin afecta los 4 lados, generando espacios no deseados en los bordes exteriores del contenedor.
Interactividad
¿Cuál es la sintaxis correcta para rotar un elemento 90 grados y escalarlo al 150% simultáneamente?
Correcto: B. La propiedad transform acepta múltiples funciones separadas por espacio. Las unidades deg son obligatorias en rotate().
Tipografía — Desarrollo
Explica la diferencia entre letter-spacing y word-spacing. Da un ejemplo de cuándo usarías cada una.
Respuesta modelo: letter-spacing controla el espacio entre cada carácter individual (útil en títulos para mayor elegancia o en texto con mayúsculas). word-spacing controla el espacio entre palabras completas, sin afectar letras dentro de una misma palabra. Ejemplo: un título de marca puede usar letter-spacing: 3px para un efecto espaciado elegante; un texto con palabras muy juntas puede usar word-spacing: 6px para mejorar la legibilidad.
Modelo de Caja — Desarrollo
¿Por qué es mejor práctica usar max-width: 100% en imágenes en lugar de width: 100%? ¿Qué problema específico soluciona?
Respuesta modelo: width: 100% fuerza a la imagen a ocupar siempre el 100% del contenedor, incluso si este es más grande que la imagen original, provocando pixelación y pérdida de calidad. max-width: 100% establece un límite máximo igual al contenedor, pero si el contenedor es más grande, la imagen conserva su tamaño natural sin distorsionarse. Si el contenedor es más pequeño, la imagen sí se reduce fluidamente, logrando un comportamiento responsivo ideal.
Flexbox
Al aplicar display: flex a un contenedor, ¿cómo se organizan sus hijos directos por defecto?
Correcto: B. Flexbox establece flex-direction: row por defecto, organizando a los hijos en fila horizontal desde el inicio (izquierda).
Interactividad
Para que un botón cambie su color de fondo de forma fluida al hacer hover, ¿qué combinación es correcta?
Correcto: C. Se necesitan ambas partes: la transition en el estado base (define qué animar y cuánto dura) + el :hover con el valor de destino. Sin transition, el cambio es instantáneo y brusco.
Integración— Desarrollo
Escribe el CSS completo para un botón con: esquinas redondeadas de 8px, sombra al hacer hover, y transición fluida de 0.3 segundos.
Respuesta modelo:

.boton {
  border-radius: 8px;
  box-shadow: none;
  transition: box-shadow 0.3s ease;
}
.boton:hover {
  box-shadow: 0px 4px 12px rgba(0,0,0,0.3);
}

Puntos clave: border-radius para redondear, la transition va en el estado base, y box-shadow en el :hover para que aparezca al pasar el cursor.
